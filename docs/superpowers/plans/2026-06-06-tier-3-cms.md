# Tier 3 Visual CMS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the portfolio editable through Sveltia CMS and automatically deploy every published content change to Cloudflare Pages.

**Architecture:** Project entries live as YAML-frontmatter Markdown files and site-wide copy lives in JSON. A Node-only Vite plugin validates and bundles that content into the existing static SPA, while stable project slugs drive the Colonist page. Sveltia CMS commits edits to GitHub, and GitHub Actions builds, tests, and deploys `dist` to the existing Pages project.

**Tech Stack:** Vite, vanilla JavaScript, YAML, Vitest, Sveltia CMS, GitHub Actions, Cloudflare Pages, Cloudflare Workers
